FAQ

What platforms can I use this API with?

This API can be accessed remotely via GET requests from the url: 

https://nickr.xyz/coolbananas/api/

The API can also be run locally using Python 3 and Django, and can be accessed similarly from the localhost: 

http://localhost:5002/coolbananas/api/
How do I use the program through the website?

To use the API through the website, access the following URL, adding your GET request terms to the end: 

https://nickr.xyz/coolbananas/api/

An example usage of this API is: 

https://nickr.xyz/coolbananas/api/?InstrumentIDs=BHP.AX,BLT.L&TopicCodes=AMERS,COM&StartDate=2015-10-01T00:00:00.000Z&EndDate=2015-10-10T00:00:00.000Z

You can easily access this using a curl call from your Bash prompt, or by visiting the page in your web browser.

How do I use the program as a standalone?

First, make sure you have a version of Python equal to newer to Python 3.5. You can get Python from here. Install this and make sure that it is added to your computer's PATH.

Second, you need to get the source files. You can get them from here. Make sure to unzip this to a folder.

Now, go to the folder that you unzipped the API to, and run this command: 

pip3 -r requirements.txt

You might need to run a different executable depending on your platform (sometimes it's just pip, sometimes you might need python pip or python3 pip).

Then, to run the program, just run: 

python3 main.py

This should start the program properly. The output should list a local URL you should use to access the program. Follow the instructions for the above section, but use a URL starting with: 

http://localhost:5002/coolbananas/api/

This might vary depending on what port the server is running on.

What are the parameters I pass in?
InstrumentIDs
 An array of all the instrument IDs that you want to know about. This is command separated with no spaces for multiple IDs. 
TopicCodes
 An array of all the topic codes that you want to know about. Also comma separated, no spaces. 
StartDate
 The date that the search starts at. This is in the format: %Y-%m-%dT%H:%M:%S.%fZ. An example is 2015-01-01T00:00:00.000Z 
EndDate
 Same as the start date, just it's the end date.
What properties should I expect?

After querying, you should receive a JSON object that you can parse. The properties are as follows: 

success : bool
 Returns whether the search was successful or not. 
error : str
 A message explaining what went wrong. This only appears if success is false. 
query_time : float
 How long the server took to generate your results. This only appears if success is true. 
NewsDataSet : []
 An array of all the results of the search. This only appears if success is true.

In addition, the objects of NewsDataSet have these properties: 

URI : str
 A link to the source of this object. 
Language : str
 The language this article is written in. 
TimeStamp : str
 When the article was written. This is in the same format as the parameter: %Y-%m-%dT%H:%M:%S.%fZ. 
Headline : str
 The headline of the article. 
NewsText : str
 The content of the article. 
InstrumentIDs : [str]
 All the instrument IDs that matched this article. 
TopicCodes : [str]
 All the topic codes that matched this article.
