FAQ

What platforms can I use this API with?

This API can be accessed remotely via GET requests from the URL:

https://nickr.xyz/coolbananas/api/

The API can also be run locally using Python 3 and Django, and can be accessed similarly from the localhost:

http://localhost:5002/coolbananas/api/

How do I use the program through the website?

To use the API through the website, access the following URL, adding your GET request terms to the end:

https://nickr.xyz/coolbananas/api/

An example usage of this API is:

https://nickr.xyz/coolbananas/api/?InstrumentIDs=BHP.AX,BLT.L&TopicCodes=AMERS,COM&StartDate=2015-10-01T00:00:00.000Z&EndDate=2015-10-10T00:00:00.000Z

You can easily access this using a curl call from your Bash prompt, or by visiting the page in your web browser.
For example, the above API would be done using the following curl command:

curl -X GET https://nickr.xyz/coolbananas/api/?InstrumentIDs=BHP.AX,BLT.L&TopicCodes=AMERS,COM&StartDate=2015-10-01T00:00:00.000Z&EndDate=2015-10-10T00:00:00.000Z

How do I use the program as a standalone?

First, make sure you have a version of Python equal to or newer than Python 3.5. You can get Python from http://python.org. Install this and make sure that it is added to your computer's PATH.

Second, you need to get the source files. You can get them from here. Make sure to unzip this to a folder.

Now, go to the folder that you unzipped the API to, and run this command:

pip3 -r requirements.txt

You might need to run a different executable depending on your platform (sometimes it's just pip, sometimes you might need python3 pip).

Then, to run the program, just run:

python3 main.py

This should start the program properly. The output should list a local URL you should use to access the program. Follow the instructions for the above section, but use a URL starting with:

http://localhost:5002/coolbananas/api/

This might vary depending on what port the server is running on. The default port is 5002, and can be changed in main.py.

What are the parameters I pass in?
Parameter Description 
InstrumentIDs
 An array of all the instrument IDs that you want to know about.
This is a comma delimited list of Reuters Instrument Codes.		 
TopicCodes
 An array of all the topic codes that you want to know about.
This is a comma delimited list of Reuters Topic Codes		 
StartDate
 The date that the search starts at.
This is in the format of %Y-%m-%dT%H:%M:%S.%fZ.
An example is 2015-10-01T00:00:00.000Z		 
EndDate
 The date that the search ends at.
This is in the format of %Y-%m-%dT%H:%M:%S.%fZ.
An example is 2015-10-10T00:00:00.000Z
What properties should I expect?

By querying, you will receive a JSON object, with the following properties:

Property Type Description 
success
 boolean Returns whether the search was successful or not. 
error
 string A message explaining what went wrong. This only appears if success is false. 
query_time
 float How long the server took to generate your results. This only appears if success is true. 
NewsDataSet
 [object] An array of all the results of the search. This only appears if success is true.

The objects of NewsDataSet have the following properties:

Property Type Description 
URI
 string A link to the source of this object. 
Language
 string The language this article is written in. 
TimeStamp
 string When the article was written.
This is in the same format as the parameter: %Y-%m-%dT%H:%M:%S.%fZ.		 
Headline
 string The headline of the article. 
NewsText
 string The content of the article. 
InstrumentIDs
 [string] All the instrument IDs that matched this article. 
TopicCodes
 [string] All the topic codes that matched this article.
